# Hello Park — Конструктор праздников: Правила разработки

## Проект
**Что это:** Веб-приложение «Конструктор праздников» для Hello Park — пошаговый Wizard (7 шагов), по которому клиент собирает свой праздник и отправляет заявку менеджеру.

**Репозиторий:** `/Users/dima/Desktop/DR-construct(Anti)/`

**Ключевые файлы:**
- `PRD.md` — бизнес-требования и user flow
- `DATA.md` — весь контент: пакеты, цены, аниматоры, МК, торты
- `src/app/App.tsx` — корень приложения, роутинг по шагам
- `src/app/components/wizard-context.tsx` — глобальный state (Context API)
- `src/app/components/step1-format.tsx` … `step7-summary.tsx` — шаги wizard'а
- `src/styles/theme.css` — дизайн-токены (цвета, радиусы, шрифты)

---

## Стек
- **React 18 + TypeScript + Vite**
- **Tailwind CSS v4** (конфиг через `@theme inline` в `theme.css`)
- **Motion/React** (Framer Motion) — все анимации
- **Lucide React** — иконки
- **Radix UI** — базовые компоненты (если нужны)
- **Figma Make** — из него импортируются ассеты через `figma:asset/...`

## Дизайн-система Hello Park

### Цвета (строго эти)
| Токен | Значение | Использование |
|-------|----------|---------------|
| `#FF6022` | Оранжевый | Основной акцент, CTA-кнопки, выделение |
| `#6C4AED` | Фиолетовый | Экселюзив-пакет, вторичный акцент |
| `#1A1A1A` | Почти чёрный | Основной текст |
| `#747474` | Серый | Вспомогательный текст |
| `#ABABAB` | Светлый серый | Подписи, неактивные элементы |
| `#E5E5E5` | Рамки | Границы карточек |
| `#F5F5F5` | Фон | Фон чипов, полей ввода |
| `#F7F7F7` | Фон страницы | `bg-background` |
| `#FFFFFF` | Белый | Карточки |

**Нельзя** использовать произвольные цвета — только из этого списка или `theme.css`.

### Шрифт
- **Gilroy** — основной шрифт бренда, всегда в теле приложения
- Fallback: `Nunito, sans-serif`

### Компоненты карточек
- `rounded-2xl` — радиус всех карточек
- `rounded-xl` — маленькие карточки (premium costumes, cakes)
- `rounded-full` — кнопки, чипы, бейджи
- Тень при ненажатом: `shadow-sm`, при выбранном: `shadow-md`
- Граница ненажатого: `ring-1 ring-[#E5E5E5]`
- Граница выбранного: `ring-2 ring-[#FF6022]` (или `ring-[#6C4AED]` для Exclusive)

### Анимации
- Все шаги используют `motion.div` с `initial={{ opacity: 0, x: 30 }}` / `exit={{ opacity: 0, x: -30 }}`
- `AnimatePresence mode="wait"` на уровне App.tsx
- `whileTap={{ scale: 0.97 }}` на кликабельных карточках
- `transition={{ duration: 0.3 }}` — стандартный переход

### Mobile First
- Максимальная ширина контента: `max-w-lg mx-auto`
- Паддинги: `px-4`
- Шрифты: `text-sm` для тела, `text-xs` для подписей
- Всё должно отлично выглядеть на экране 390px (iPhone 14)

## Правила кода

### Структура
- **Данные (массивы с контентом) — только в `DATA.md` и `src/data/`**, не инлайн в компонентах
- Каждый шаг wizard'а — отдельный файл `step{N}-name.tsx`
- State только через `wizard-context.tsx` — не заводить локальный state для данных формы
- Переиспользуемые UI-детали → `src/app/components/ui/`

### Стиль кода
- Tailwind-классы: использовать токены через `theme.css`, а не хардкодить цвета (`text-primary`, `bg-background` и т.д.)
- Исключение: `[#FF6022]` и `[#6C4AED]` допустимы, если токена нет
- Не использовать `inline style` без крайней необходимости
- Нет `any` в TypeScript

### Навигация между шагами
- Кнопки «Назад» / «Далее» — через FloatingPrice (нижняя плавающая панель)
- `nextStep()` и `prevStep()` из `useWizard()`

## Бизнес-логика

### Пакеты и цены
Смотри `DATA.md` — единственный источник правды.

### Правило включённости опций в пакет
- **basic (Hello Fun):** без пати-рума, без МК, без еды в пакете
- **premium (Hello Party):** пати-рум включён, без МК, без еды
- **exclusive (Hello Bloom):** пати-рум + 1 МК включены, без еды
- **custom (Собери свой):** всё считается отдельно

### Цены зависимые от дня
- Депозиты за кафе-зону: будни — меньше, выходные — больше
- Фиксируется при выборе даты на шаге 6

## Что НЕ делать
- ❌ Не использовать сторонние UI-библиотеки (Material UI, Ant Design) — только Radix + Lucide
- ❌ Не добавлять Zustand/Redux — state управляется через Context
- ❌ Не создавать лишних файлов — держать структуру чистой
- ❌ Не хардкодить тексты прямо в компонентах — данные в `DATA.md` → `src/data/`
- ❌ Не забывать про мобильную адаптацию
